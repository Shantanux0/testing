SkillSwap:
The Mathematics of Matching

Model: Multi-Objective Weighted Sum
Objective: Heuristic Optimization
Calculated In: Java Engine

1 The Core Scoring Algorithm

Our internal AI uses a Multi-Objective Decision Making (MODM) approach. The final compatibility between any two peers is defined by the following summation:

FinalScore = Σ (ni * wi)

Σ   = The Summation of all matching factors

ni = The Normalized Input of factor i (0.0 to 1.0)

wi = The Heuristic Weight assigned to factor i

How it works:

1. Metric Collection

We extract diverse data points: Test Scores from Groq AI, Experience from Resumes, and Goals from User Profiles.

2. Unit Normalization

Since 'Years of Experience' and 'Test Score' use different units, we normalize them to a standard 0-1 scale using Min-Max Scaling.

3. Heuristic Priority

Each factor is multiplied by its Weight (wi). High-priority factors like Verified Expertise carry more influence than secondary factors.

4. Final Summation

The weighted signals are summed. A score > 0.8 is flagged as a 'Perfect Swap', triggering immediate email notifications.
